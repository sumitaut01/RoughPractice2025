1 The Three Pillars of Git

Git isn't just a "save" button; it's a three-stage management system.

Working Directory

Where you edit files. Files are untracked or modified.

git add <file>

Staging Area (Index)

The "Prepare to Commit" zone. Marks changes for the next snapshot.

git commit -m "..."

HEAD (Repository)

The official record. HEAD points to your current location in history.

git log

2 The "Sync" Confusion: Fetch vs Pull

git fetch (Safe / Inspect)

Downloads new commits/branches but does not touch your files. Updates origin/main.

"Checking your mail without opening the envelopes."

git pull (Aggressive / Merge)

Fetches + Merges automatically. Modifies your files immediately.

"Downloading and applying the changes right now."

Pro Tip:

Always fetch first if you want to see what others did before risking a merge conflict in your local code.

3 Branching & "Detached HEAD"

Creating a Trail

When you branch from main, your new branch inherits all previous logs. Git stores commit ancestry, not copies of files.

git checkout -b new-feature
# Now new-feature log == main log

Detached HEAD

Happens when you checkout a Commit Hash instead of a branch.

  • You are no longer on a branch.
  • Commits made here are orphaned (lost) unless you create a branch.
  • To fix: git checkout -b fix-state

4 The "Undo" War: Reset vs Revert

Feature git reset git revert
Modifies History? Yes (Erase commits) No (Adds new commit)
Safe for Remote? ❌ Dangerous ✅ Safe
Best for... Local cleanup before push Undoing public mistakes
Reset --hard

Destroys everything in staging and working directory. Returns files to exactly how they were at that commit.

Restore --staged

Unstages files but keeps your edits. Moves them from Staging back to Working Dir.

5 Pull Request: Internal Operations

Why we never "Pull origin feature" directly into master.

1

Diff Generation

GitHub compares main vs feature line-by-line.

2

CI/CD Gatekeeping

Pipelines run tests automatically. Merging is blocked if tests fail.

3

Audit Trail

Direct pulls destroy the record of who approved and what conversations happened.

The Danger of Direct Pull

If you git pull origin feature into master, you skip all protection rules. You might accidentally merge broken code or half-finished features that haven't been reviewed.

6 Professional Merging Workflow

Industry Standard (Merge-Based)

Step 1

Sync Main → Feature locally first.

git checkout feature/login && git merge main
Step 2

Resolve conflicts locally.

No surprises during the PR process!

Step 3

Push updated feature & Create PR.

Merge Commit

Full history preserved. Good for debugging. Safe.

Squash & Merge

Clean main branch. Combines 10 feature commits into 1.

Rebase & Merge

Linear history. No merge bubbles. Expert level.
