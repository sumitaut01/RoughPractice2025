Base Test Class

Set up everything once in a base class. All test classes extend this.

java ¬∑ BaseTest.java 
public class BaseTest {

    protected RequestSpecification requestSpec;
    protected ResponseSpecification responseSpec;

    @BeforeClass
    public void setup() {
        // Global ObjectMapper config
        ObjectMapper mapper = new ObjectMapper();
        mapper.registerModule(new JavaTimeModule());
        mapper.configure(DeserializationFeature.FAIL_ON_UNKNOWN_PROPERTIES, false);
        mapper.setPropertyNamingStrategy(PropertyNamingStrategies.SNAKE_CASE);

        RestAssured.config = RestAssuredConfig.config()
            .objectMapperConfig(ObjectMapperConfig.objectMapperConfig()
                .jackson2ObjectMapperFactory((cls, charset) -> mapper));

        // Request spec
        requestSpec = new RequestSpecBuilder()
            .setBaseUri("https://api.example.com")
            .setContentType(ContentType.JSON)
            .addHeader("Authorization", "Bearer " + TokenManager.getToken())
            .addFilter(new AllureRestAssured())
            .log(LogDetail.ALL)
            .build();

        // Response spec
        responseSpec = new ResponseSpecBuilder()
            .expectStatusCode(200)
            .expectContentType(ContentType.JSON)
            .build();
    }
}

Serialization & Deserialization

Deserialization JSON ‚Üí Java

  • Response JSON mapped into a POJO
  • Use .extract().as(User.class)
  • Jackson matches JSON fields to POJO fields
  • Use TypeRef for generic types like Lists

Serialization Java ‚Üí JSON

  • POJO converted to JSON for request body
  • Use .body(myObject)
  • Must set ContentType.JSON
  • Can also use Map instead of POJO

POJO for Nested Response

java ¬∑ User.java (POJO)

Jackson Annotations

üõ°Ô∏è

@JsonIgnoreProperties

Silently ignore unknown fields in response. Always use this ‚Äî APIs add new fields constantly.

üî§

@JsonNaming

Apply naming strategy to all fields in the class. Use SnakeCaseStrategy for APIs that return snake_case.

üè∑Ô∏è

@JsonProperty

Map a specific JSON field name to a Java field. Overrides @JsonNaming for that field.

üö´

@JsonIgnore

Skip a specific field during serialization or deserialization ‚Äî passwords, internal fields, etc.

üìÖ

@JsonFormat

Define the date/time pattern for a field. Required for LocalDateTime, LocalDate.

üßπ

@JsonInclude

Skip null fields when serializing to JSON. Use NON_NULL to keep request bodies clean.

java ¬∑ All annotations in one POJO 
@JsonIgnoreProperties(ignoreUnknown = true)          // ignore extra fields
@JsonNaming(PropertyNamingStrategies.SnakeCaseStrategy.class)  // first_name ‚Üí firstName
@JsonInclude(JsonInclude.Include.NON_NULL)             // skip nulls in requests
public class User {

    private int id;

    private String firstName;   // auto-mapped from "first_name"

    @JsonProperty("email_address")  // explicit override
    private String email;

    @JsonIgnore                      // never serialized or deserialized
    private String password;

    @JsonFormat(pattern = "yyyy-MM-dd'T'HH:mm:ss")
    private LocalDateTime createdAt;
}
üí°

Pro tip: @JsonProperty always wins over @JsonNaming. Set a global naming strategy on the class, then override specific non-conforming fields with @JsonProperty.

Validation + TestNG

Approach 1 ‚Äî Inline Hamcrest (Quick Checks)

java 
given().spec(requestSpec)
.when().get("/users/1")
.then()
    .statusCode(200)
    .body("name",               equalTo("Leanne Graham"))
    .body("address.city",       equalTo("Gwenborough"))     // nested
    .body("address.geo.lat",    notNullValue())              // deep nested
    .body("$",                  hasSize(10))                 // list size
    .body("name",               hasItem("Leanne Graham"))    // list contains
    .body("[0].address.city",   equalTo("Gwenborough"));    // first item nested

Approach 2 ‚Äî POJO + TestNG Assert

java

    .when().get("/users/1")
    .then().extract().as(User.class);

Assert.assertEquals(user.getName(), "Leanne Graham");
Assert.assertNotNull(user.getAddress());
Assert.assertEquals(user.getAddress().getCity(), "Gwenborough");
Assert.assertFalse(user.getOrders().isEmpty());

Approach 3 ‚Äî SoftAssert ‚≠ê Recommended

‚ö†Ô∏è

Without SoftAssert, the test stops at the first failure. You won't know about other failures until you fix the first one. Always use SoftAssert for API field validations.

java 
@Test
public void validateUser() {
    User user = given().spec(requestSpec)
        .when().get("/users/1")
        .then().extract().as(User.class);

    SoftAssert soft = new SoftAssert();

    soft.assertEquals(user.getId(),                  1,             "ID mismatch");
    soft.assertEquals(user.getName(),               "Leanne Graham", "Name mismatch");
    soft.assertNotNull(user.getAddress(),                            "Address null");
    soft.assertEquals(user.getAddress().getCity(), "Gwenborough",   "City mismatch");
    soft.assertFalse(user.getOrders().isEmpty(),                     "Orders empty");

    soft.assertAll();  // MUST call ‚Äî reports ALL failures at once
}

Chaining Requests

The response of one API call becomes the input to the next ‚Äî critical for end-to-end flows.

01
Login
‚Üí get token
‚Üí
02
Create User
‚Üí get userId
‚Üí
03
Create Order
‚Üí get orderId
‚Üí
04
Verify Order
‚Üí assertions
‚Üí
05
Cleanup
‚Üí delete
java ¬∑ Full chained flow 
@Test
public void fullOrderFlow() {

    // Step 1 ‚Äî Create user, capture userId
    int userId = given().spec(requestSpec)
        .body(Map.of("name", "John", "email", "john@test.com"))
      .when().post("/users")
      .then().statusCode(201)
        .extract().jsonPath().getInt("data.id");

    // Step 2 ‚Äî Create order using userId
    int orderId = given().spec(requestSpec)
        .body(Map.of("user_id", userId, "item", "Laptop", "amount", 999.99))
      .when().post("/orders")
      .then().statusCode(201)
        .extract().jsonPath().getInt("data.order_id");

    // Step 3 ‚Äî Verify order using orderId
    SoftAssert soft = new SoftAssert();
    Order order = given().spec(requestSpec)
      .when().get("/orders/" + orderId)
      .then().statusCode(200)
        .extract().as(Order.class);

    soft.assertEquals(order.getUserId(), userId);
    soft.assertEquals(order.getItem(), "Laptop");

    // Step 4 ‚Äî Cleanup
    given().spec(requestSpec)
      .when().delete("/orders/" + orderId)
      .then().statusCode(204);

    soft.assertAll();
}

Auth Types Deep Dive

Basic Auth

Sends username:password encoded in Base64 on every request. NOT encryption ‚Äî just encoding. Safe only over HTTPS.

Credentials:
john : secret123
Base64 encoded:
am9objpzZWNyZXQxMjM=
Header sent:
Authorization: Basic am9objpzZWNyZXQxMjM=
java
given().auth().basic("john", "secret123")

Digest Auth

Password never travels over the wire. Server sends a one-time nonce (challenge), client hashes the password + nonce and sends the hash. Server does the same hash and compares.

01
Request
no credentials
‚Üí
02
401 + nonce
server challenge
‚Üí
03
Hash(pass+nonce)
client computes
‚Üí
04
Send hash
not password!
‚Üí
05
200 OK
server validates
java
given().auth().digest("john", "secret123")

Bearer Token / JWT

Login once, get a token, use it everywhere. The JWT token carries user identity AND permissions inside its payload ‚Äî no DB lookup needed on each request.

java ¬∑ JWT structure
// JWT = header.payload.signature
eyJhbGciOiJIUzI1NiJ9 . eyJ1c2VySWQiOjEsInJvbGUiOiJhZG1pbiIsImV4cCI6MTcwNX0 . abc123sig

// Decoded payload:
{
  "userId": 1,
  "role":   "admin",        // authorization info
  "perms": ["read", "write", "delete"],
  "exp":    1705312200       // expiry
}

OAuth2

Delegated access ‚Äî let a third-party app access your resources without giving it your password. Scopes define exactly what is permitted.

java ¬∑ Client Credentials flow (machine-to-machine)
// Step 1 ‚Äî Get token using client credentials
String token = given()
    .contentType(ContentType.URLENC)
    .formParam("grant_type",    "client_credentials")
    .formParam("client_id",     System.getenv("CLIENT_ID"))
    .formParam("client_secret", System.getenv("CLIENT_SECRET"))
    .formParam("scope",         "read:users write:orders")
  .when().post("/oauth/token")
  .then().statusCode(200)
    .extract().jsonPath().getString("access_token");

// Step 2 ‚Äî Use like Bearer token
given().auth().oauth2(token)

Token Manager ‚Äî Dynamic Token Handling

java ¬∑ TokenManager.java
public class TokenManager {
    private static String accessToken;
    private static long tokenExpiryTime;
    private static final String USERNAME = System.getenv("API_USERNAME");
    private static final String PASSWORD = System.getenv("API_PASSWORD");

    public static synchronized String getToken() {
        if (accessToken == null || System.currentTimeMillis() > tokenExpiryTime) {
            fetchNewToken();
        }
        return accessToken;
    }

    private static void fetchNewToken() {
        Response res = given()
            .contentType(ContentType.JSON)
            .body(Map.of("username", USERNAME, "password", PASSWORD))
          .when().post("/auth/login")
          .then().statusCode(200).extract().response();

        accessToken = res.jsonPath().getString("data.access_token");
        int expiresIn = res.jsonPath().getInt("data.expires_in");
        tokenExpiryTime = System.currentTimeMillis() + (expiresIn * 1000L) - 5000L;
    }

    // For role-based testing
    public static String getToken(String role) { /* ... */ }
}

Authentication vs Authorization

üîë Authentication

"Who are you?"

  • Verifying identity
  • Username + password check
  • Token validation
  • Happens first
  • Failure ‚Üí 401 Unauthorized

üõ°Ô∏è Authorization

"What can you do?"

  • Checking permissions
  • Role-based access control
  • Scope validation
  • Happens after authentication
  • Failure ‚Üí 403 Forbidden
üè¢

Analogy: Authentication = Security checks your ID at the office entrance. Authorization = Your ID only gives you access to floors 1 and 2, not the CEO's floor 3.

401
Unauthorized
Authentication failed. No token, invalid token, or expired token.
403
Forbidden
Authentication passed but Authorization failed. Valid token, wrong role/permission.
200
OK
Both authentication and authorization passed.

Testing Both in Code

java ¬∑ Auth negative tests
// 401 ‚Äî Authentication failure (no token)
@Test
public void noToken_returns401() {
    given().baseUri("https://api.example.com")
      .when().get("/secure/users")
      .then().statusCode(401);
}

// 401 ‚Äî Authentication failure (invalid token)
@Test
public void invalidToken_returns401() {
    given().header("Authorization", "Bearer invalidtoken123")
      .when().get("/secure/users")
      .then().statusCode(401);
}

// 403 ‚Äî Authorization failure (valid token, wrong role)
@Test
public void customerCannotDelete_returns403() {
    given().header("Authorization", "Bearer " + TokenManager.getToken("customer"))
      .when().delete("/users/5")
      .then().statusCode(403);  // knows who you are, but NO permission
}

JSON Schema Validation

Instead of validating field by field, schema validation checks the entire response structure ‚Äî field names, types, required fields ‚Äî in one shot.

json ¬∑ src/test/resources/schemas/user-schema.json
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "type": "object",
  "required": ["id", "name", "email", "address"],   // must be present
  "properties": {
    "id":    { "type": "integer" },
    "name":  { "type": "string"  },
    "email": { "type": "string", "format": "email" },
    "address": {
      "type": "object",
      "required": ["street", "city"],
      "properties": {
        "street": { "type": "string" },
        "city":   { "type": "string" },
        "geo": {
          "type": "object",
          "properties": {
            "lat": { "type": "string" },
            "lng": { "type": "string" }
          }
        }
      }
    },
    "orders": {
      "type": "array",
      "items": {
        "required": ["order_id", "item", "amount"],
        "properties": {
          "order_id": { "type": "integer" },
          "item":     { "type": "string"  },
          "amount":   { "type": "number"  },
          "status":   { "type": "string"  }
        }
      }
    }
  }
}
java ¬∑ Using schema validation
import static io.restassured.module.jsv.JsonSchemaValidator.matchesJsonSchemaInClasspath;

@Test
public void validateSchema() {
    given().spec(requestSpec)
      .when().get("/users/1")
      .then()
        .statusCode(200)
        .body(matchesJsonSchemaInClasspath("schemas/user-schema.json"))  // one line!

}
