01 · FUNDAMENTALS

What IS a Stream?

A stream is a pipeline of operations on a sequence of elements — not a data structure, but a way to process one.

1
🏭
Factory Conveyor Belt
Raw materials (data) flow through stations (operations). Each station transforms them. At the end, you collect the final product.
2
🚰
Water Pipeline
Water flows through pipes. Each pipe filters or transforms it. You open the tap (terminal op) to get the result.
3
📜
SQL for Collections
SELECT name FROM users WHERE age > 18 ≡ .filter().map().collect()

BEFORE STREAMS The old for-loop way

List<String> names = Arrays.asList("Alice", "Bob", "Charlie", "Dave");

// Old way: verbose, hard to read, mutable state
List<String> result = new ArrayList<>();
for (String name : names) {
    if (name.length() > 3) {
        result.add(name.toUpperCase());
    }
}
// result = [ALICE, CHARLIE, DAVE]

WITH STREAMS Clean, readable, declarative

List<String> names = Arrays.asList("Alice", "Bob", "Charlie", "Dave");

// Stream way: read like English, no mutation
List<String> result = names.stream()              // 1. CREATE stream
    .filter(name -> name.length() > 3)            // 2. INTERMEDIATE
    .map(String::toUpperCase)                       // 3. INTERMEDIATE
    .collect(Collectors.toList());                  // 4. TERMINAL

// result = [ALICE, CHARLIE, DAVE]
🧠

Memory Hack: Always 3 parts — SOURCE → INTERMEDIATE (zero or more) → TERMINAL (exactly one)

CREATING STREAMS All the ways

// 1. From a Collection (most common)
list.stream();
list.parallelStream();              // multi-thread version

// 2. From an Array
Arrays.stream(new int[]{1,2,3});
Stream.of("a", "b", "c");

// 3. From a range (IntStream)
IntStream.range(1, 6);              // 1,2,3,4,5
IntStream.rangeClosed(1, 5);       // 1,2,3,4,5

// 4. Infinite stream (use with limit!)
Stream.iterate(0, n -> n + 2).limit(5);  // 0,2,4,6,8
Stream.generate(Math::random).limit(3); // 3 random doubles

// 5. From String
"hello".chars()                       // IntStream of char values
Pattern.compile(",").splitAsStream("a,b,c") // Stream of a,b,c

03 · OPERATIONS

Every Operation, Explained

Two types: Intermediate (lazy, return a stream) and Terminal (eager, trigger execution).

⚙️ Intermediate Operations

Return a new Stream. LAZY — nothing runs until terminal is called.

filter map flatMap sorted distinct limit skip peek mapToInt

🏁 Terminal Operations

Trigger the pipeline. EAGER — runs everything. Stream is consumed after this.

collect forEach reduce count findFirst anyMatch allMatch min/max toArray
🔍
filter(Predicate)
INTERMEDIATE
Keeps only elements matching the predicate. Like a bouncer at a door.
.filter(n -> n > 5)
.filter(String::isEmpty)
.filter(e -> e.getSalary() > 50000)
🔄
map(Function)
INTERMEDIATE
Transforms each element to something else. 1 in → 1 out (shape changes).
.map(n -> n * 2)
.map(String::toUpperCase)
.map(Employee::getName)
📂
flatMap(Function)
INTERMEDIATE
Flattens nested streams. 1 in → N out. Ideal for lists of lists.
// List<List<Int>> → Stream<Int>
.flatMap(Collection::stream)
.flatMap(s -> Arrays.stream(s.split(" ")))
🧮
reduce(identity, BinaryOp)
TERMINAL
Combines all elements into ONE value. Like a snowball rolling.
.reduce(0, (a,b) -> a+b)  // sum
.reduce(1, (a,b) -> a*b)  // product
.reduce(Integer::max)     // max value
🎁
collect(Collector)
TERMINAL
Mutable reduction — packages stream into List, Map, Set, String, etc.
.collect(Collectors.toList())
.collect(Collectors.toSet())
.collect(Collectors.joining(", "))
🎯
findFirst / anyMatch
TERMINAL
Short-circuit ops — stop as soon as condition is met. Very efficient!
.findFirst()            // Optional<T>
.anyMatch(n -> n>10)  // boolean
.allMatch(n -> n>0)   // boolean

REDUCE EXPLAINED The Snowball Mental Model

reduce starts with an identity value and rolls over each element, accumulating:

// reduce(0, (acc, n) -> acc + n) on [1, 2, 3, 4, 5]
04 · COLLECTORS

Collectors — The Packaging Plant

After processing, Collectors package the result. Think of them as different types of boxes.

Collector Returns Use Case Example
toList() List<T> Most common — ordered, allows duplicates collect(toList())
toSet() Set<T> Unique elements only collect(toSet())
toMap() Map<K,V> Key-value pairs from stream toMap(k::id, k::name)
joining() String Concatenate strings joining(", ", "[", "]")
groupingBy() Map<K, List<V>> SQL GROUP BY equivalent groupingBy(Employee::getDept)
partitioningBy() Map<Boolean, List> Split into true/false groups partitioningBy(n -> n > 0)
counting() Long Count elements per group groupingBy(k, counting())
summingInt() Integer Sum a numeric field summingInt(Employee::getAge)

groupingBy — The most powerful collector

List<Employee> employees = /* ... */;

// Group employees by department
Map<String, List<Employee>> byDept =
    employees.stream()
             .collect(Collectors.groupingBy(Employee::getDepartment));

// Count per department
Map<String, Long> countByDept =
    employees.stream()
             .collect(Collectors.groupingBy(
                 Employee::getDepartment,
                 Collectors.counting()
             ));

// Average salary per department
Map<String, Double> avgSalary =
    employees.stream()
             .collect(Collectors.groupingBy(
                 Employee::getDepartment,
                 Collectors.averagingDouble(Employee::getSalary)
             ));

joining() — String assembly

List<String> names = ...;

// Simple join
names.stream()
     .collect(Collectors.joining());
// "AliceBobCharlie"

// With delimiter
.collect(Collectors.joining(", "));
// "Alice, Bob, Charlie"

// With prefix+suffix
.collect(Collectors.joining(", ", "[", "]"));
// "[Alice, Bob, Charlie]"

toMap() — Build a lookup map

// employees → Map<id, name>
Map<Integer, String> idToName =
    employees.stream()
             .collect(Collectors.toMap(
                 Employee::getId,   // key
                 Employee::getName  // value
             ));

// Handle duplicates with merge
.collect(Collectors.toMap(
    Employee::getDept,
    Employee::getSalary,
    Double::max // keep higher salary
));
05 · LAZY EVALUATION

Lazy Evaluation — The Secret Superpower

Streams don't process anything until a terminal operation is called. This is the key to their efficiency.

🎮 LAZY DEMO — Watch the execution order

List<Integer> nums = Arrays.asList(1,2,3,4,5);

nums.stream()
    .filter(n -> { System.out.println("filter: " + n); return n % 2 == 0; })
    .map(n -> { System.out.println("map: " + n); return n * 10; })
    .findFirst(); // ← terminal, triggers execution
// Click "Run" to see lazy execution order...
🧠

Key Insight: Elements are processed one-by-one through the pipeline, not stage-by-stage! Element 1 goes through ALL stages before element 2 starts. findFirst() stops as soon as it finds the first match — it won't process 3, 4, 5 at all!

Execution Order: Vertical (element-by-element), NOT Horizontal (stage-by-stage)

❌

Wrong Mental Model:
All filter → then all map → then all collect

✅

Correct Mental Model:
elem1 through all stages → elem2 through all stages → ...

// Actual execution order for [1,2,3]:
// filter(1)→false | filter(2)→true | map(2)→20 | filter(3)→false
// Result: [20]    ← only 2 was mapped! 1 and 3 never reached map()

// THIS is why streams can be faster than loops for large data +
// short-circuit operations like findFirst(), anyMatch()
06 · ADVANCED

Advanced Patterns

FlatMap, Optional, parallel streams, method references, and real-world chaining.

FLATMAP — Flatten nested structures

// Problem: List of lists → you want a flat stream
List<List<Integer>> nested = Arrays.asList(
    Arrays.asList(1, 2, 3),
    Arrays.asList(4, 5),
    Arrays.asList(6, 7, 8)
);

// map gives Stream<Stream<Int>> — WRONG
nested.stream().map(Collection::stream); // ❌

// flatMap gives Stream<Int> — CORRECT
nested.stream()
      .flatMap(Collection::stream)          // ✅
      .collect(Collectors.toList());
// [1, 2, 3, 4, 5, 6, 7, 8]

// Real world: all words from sentences
List<String> sentences = Arrays.asList("Hello World", "Java Streams");
sentences.stream()
         .flatMap(s -> Arrays.stream(s.split(" ")))
         .collect(Collectors.toList());
// [Hello, World, Java, Streams]
🧠

Memory Hack — map vs flatMap:
map = wraps in a box → you get a box of boxes 📦📦
flatMap = opens each box, empties contents into one big box 📭

OPTIONAL — Safe null handling with streams

// findFirst() returns Optional — never null!
Optional<Employee> found = employees.stream()
    .filter(e -> e.getName().equals("Sumit"))
    .findFirst();

// Bad way
found.get();                         // ❌ throws if empty

// Good ways
found.isPresent();                    // check first
found.orElse(new Employee("Default")); // fallback value
found.orElseGet(() -> fetchFromDB());  // lazy fallback
found.orElseThrow(() -> new RuntimeException("Not found"));
found.ifPresent(e -> System.out.println(e.getName()));

PARALLEL STREAMS — Multi-core power

// Just change .stream() to .parallelStream()
long sum = IntStream.rangeClosed(1, 1_000_000)
                    .parallel()       // uses ForkJoinPool
                    .sum();

// Or from collection
list.parallelStream()
    .filter(n -> n % 2 == 0)
    .collect(Collectors.toList());
⚠️

Parallel != Always Faster. Use for CPU-heavy operations on large datasets (>10k elements). Avoid for I/O, small collections, or operations with shared mutable state — these cause race conditions!

METHOD REFERENCES — 4 types, clean code

// Type 1: Static method   ClassName::staticMethod
.map(Integer::parseInt)      // same as s -> Integer.parseInt(s)

// Type 2: Instance method on type   ClassName::instanceMethod
.map(String::toUpperCase)   // same as s -> s.toUpperCase()
.filter(String::isEmpty)    // same as s -> s.isEmpty()

// Type 3: Instance method on object   object::instanceMethod
.forEach(System.out::println) // same as s -> System.out.println(s)

// Type 4: Constructor   ClassName::new
.map(Employee::new)           // same as s -> new Employee(s)

🌍 Real-World Example — Employee Analytics

// Complete real-world stream chain
Map<String, Optional<Employee>> topEarnerByDept =
    employees.stream()
             .filter(e -> e.isActive())           // active only
             .filter(e -> e.getAge() >= 21)        // adults
             .sorted(Comparator
                 .comparingDouble(Employee::getSalary)
                 .reversed())                        // highest first
             .collect(Collectors.groupingBy(
                 Employee::getDepartment,
                 Collectors.reducing(BinaryOperator.minBy(
                     Comparator.comparingDouble(Employee::getSalary)
                 ))
             ));

🧠 The 5 Rules to Never Forget

RULE 1 — 3 Parts Always
Every stream has: Source → Intermediate (0 or more) → Terminal (exactly 1)
RULE 2 — Lazy Until Terminal
Nothing runs until you call a terminal op. No terminal = no work done!
RULE 3 — One-Time Use
A stream is consumed after the terminal op. Calling stream() again gives you a fresh one.
RULE 4 — Non-Interfering
Never modify the source collection while streaming it. This causes ConcurrentModificationException.
