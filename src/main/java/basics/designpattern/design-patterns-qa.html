Page Object Model (POM)
structural · selenium · testng
Intent: Encapsulate every page's locators and interactions into a dedicated class. Tests only call high-level methods like loginPage.login(user, pass) — no XPaths leak into test code. Locator changes require a fix in exactly ONE place.
BasePage.java Java · Selenium 
public class BasePage {

    protected WebDriver driver;
    protected WebDriverWait wait;

    public BasePage(WebDriver driver) {
        this.driver = driver;
        this.wait = new WebDriverWait(driver, Duration.ofSeconds(10));
        PageFactory.initElements(driver, this);
    }

    protected void click(WebElement el) {
        wait.until(ExpectedConditions.elementToBeClickable(el)).click();
    }

    protected void type(WebElement el, String text) {
        wait.until(ExpectedConditions.visibilityOf(el)).clear();
        el.sendKeys(text);
    }
}
LoginPage.java Java · Selenium 
public class LoginPage extends BasePage {

    @FindBy(id = "username")
    private WebElement usernameInput;

    @FindBy(id = "password")
    private WebElement passwordInput;

    @FindBy(css = "button[type='submit']")
    private WebElement loginBtn;

    @FindBy(css = ".error-msg")
    private WebElement errorMsg;

    public LoginPage(WebDriver driver) { super(driver); }

    // Returns next page on success — fluent chaining
    public DashboardPage login(String user, String pass) {
        type(usernameInput, user);
        type(passwordInput, pass);
        click(loginBtn);
        return new DashboardPage(driver);
    }

    public String getError() {
        return wait.until(
            ExpectedConditions.visibilityOf(errorMsg)).getText();
    }
}

01
Singleton Pattern
creational · thread-safe · driver management · config
Intent: Ensure a class has only one instance and provide a global access point to it. In QA automation: WebDriver per thread, DB connection, Config loader, Token cache.

⚠ Regular Singleton breaks parallel tests. Use ThreadLocal Singleton for TestNG parallel execution.

DriverManager.java — ThreadLocal Singleton Java · Selenium · TestNG Parallel 
public class DriverManager {

    // ThreadLocal ensures each thread gets its own WebDriver
    private static final ThreadLocal<WebDriver> driverHolder
        = new ThreadLocal<>();

    private DriverManager() {} // prevent instantiation

    public static WebDriver getDriver() {
        if (driverHolder.get() == null) {
            String browser = ConfigManager.get("browser");
            driverHolder.set(DriverFactory.create(browser)); // Factory pattern!
        }
        return driverHolder.get();
    }

    public static void quitDriver() {
        if (driverHolder.get() != null) {
            driverHolder.get().quit();
            driverHolder.remove(); // critical — avoid memory leak
        }
    }
}
ConfigManager.java — Singleton Config Loader Java 
public class ConfigManager {

    private static ConfigManager instance;
    private final Properties props = new Properties();

    private ConfigManager() {
        String env = System.getProperty("env", "qa");
        try (InputStream is = getClass().getClassLoader()
            .getResourceAsStream("config/" + env + ".properties")) {
            props.load(is);
        } catch (IOException e) {
            throw new RuntimeException("Config load failed: " + env, e);
        }
    }

    public static synchronized ConfigManager getInstance() {
        if (instance == null) instance = new ConfigManager();
        return instance;
    }

    public static String get(String key) {
        return getInstance().props.getProperty(key);
    }
}

// Usage: ConfigManager.get("base.url") → "https://qa.x4v.com/api"
TokenCache.java — Singleton for RestAssured Auth Token Java · RestAssured 
public class TokenCache {

    private static volatile TokenCache instance; // DCL Singleton
    private String token;
    private long expiresAt;

    private TokenCache() {}

    public static TokenCache getInstance() {
        if (instance == null) {
            synchronized (TokenCache.class) {
                if (instance == null) instance = new TokenCache();
            }
        }
        return instance;
    }

    public String getToken() {
        if (token == null || isExpired()) refreshToken();
        return token;
    }

    private void refreshToken() {
        token = RestAssured.given()
            .contentType("application/json")
            .body("""{"username":"admin","password":"secret"}""")
            .post(ConfigManager.get("auth.url"))
            .jsonPath().getString("token");
        expiresAt = System.currentTimeMillis() + 3600_000L;
    }

    private boolean isExpired() {
        return System.currentTimeMillis() > expiresAt;
    }
}

Real usage in your framework

  • DriverManager.getDriver() → used inside every BasePage constructor
  • ConfigManager.get("base.url") → auto-switches QA/UAT via -Denv=uat Maven arg
  • TokenCache.getInstance().getToken() → reused across all RestAssured tests without re-login
  • DBConnectionManager (similar pattern) → single JDBC connection pool per test run
02
Factory Method Pattern
creational · browser/driver instantiation · environment switching
Intent: Define an interface for creating an object, but let subclasses or a factory class decide which class to instantiate. Perfect for browser selection (Chrome/Firefox/Edge) and environment-aware base URIs for RestAssured.
DriverFactory.java Java · Selenium 
public class DriverFactory {

    public static WebDriver create(String browser) {
        return switch (browser.toLowerCase()) {
            case "chrome"  -> buildChrome();
            case "firefox" -> buildFirefox();
            case "edge"    -> buildEdge();
            default        -> throw new IllegalArgumentException(
                                    "Unknown browser: " + browser);
        };
    }

    private static WebDriver buildChrome() {
        ChromeOptions opts = new ChromeOptions();
        opts.addArguments("--headless=new", "--no-sandbox",
                          "--disable-dev-shm-usage");
        return new ChromeDriver(opts);
    }

    private static WebDriver buildFirefox() {
        FirefoxOptions opts = new FirefoxOptions();
        opts.addArguments("-headless");
        return new FirefoxDriver(opts);
    }

    private static WebDriver buildEdge() {
        return new EdgeDriver(new EdgeOptions());
    }
}

// Run: mvn test -Dbrowser=firefox -Denv=uat
ApiClientFactory.java — RestAssured Base URI factory Java · RestAssured 
public class ApiClientFactory {

    public static RequestSpecification getSpec(String service) {
        String baseUrl = switch (service) {
            case "pricing"   -> ConfigManager.get("pricing.url");
            case "auth"      -> ConfigManager.get("auth.url");
            case "inventory" -> ConfigManager.get("inventory.url");
            default          -> throw new IllegalArgumentException(service);
        };

        return RestAssured.given()
            .baseUri(baseUrl)
            .contentType(ContentType.JSON)
            .header("Authorization", "Bearer "
                + TokenCache.getInstance().getToken())
            .log().ifValidationFails();
    }
}

03
Builder Pattern
creational · request payload · test data objects · json
Intent: Construct complex objects step by step. Separate construction from representation. Ideal for building API request payloads, test data objects from JSON/Excel, and fluent RestAssured request builders.
PricingRuleRequest.java — Builder for X4V pricing API Java · RestAssured · JSON 
public class PricingRuleRequest {

    private final String  ruleCode;
    private final String  vehicleType;
    private final int     powerRank;
    private final double  basePrice;
    private final boolean isActive;
    private final List<String> tags;

    private PricingRuleRequest(Builder b) {
        this.ruleCode    = b.ruleCode;
        this.vehicleType = b.vehicleType;
        this.powerRank   = b.powerRank;
        this.basePrice   = b.basePrice;
        this.isActive    = b.isActive;
        this.tags        = b.tags;
    }

    public static class Builder {
        private String  ruleCode;
        private String  vehicleType  = "EV";       // sensible defaults
        private int     powerRank    = 1;
        private double  basePrice    = 0.0;
        private boolean isActive     = true;
        private List<String> tags = Collections.emptyList();

        public Builder ruleCode(String v)     { this.ruleCode    = v; return this; }
        public Builder vehicleType(String v)  { this.vehicleType = v; return this; }
        public Builder powerRank(int v)       { this.powerRank   = v; return this; }
        public Builder basePrice(double v)    { this.basePrice   = v; return this; }
        public Builder inactive()             { this.isActive    = false; return this; }
        public Builder tags(List<String> v) { this.tags        = v; return this; }

        public PricingRuleRequest build() {
            Objects.requireNonNull(ruleCode, "ruleCode is required");
            return new PricingRuleRequest(this);
        }
    }

    // Jackson / Gson serialize this to JSON automatically
}

public class TestDataBuilder {

    private static final ObjectMapper MAPPER = new ObjectMapper();

    // Load from src/test/resources/testdata/pricing_rules.json
    public static <T> T fromJson(String path, Class<T> type) {
        try (InputStream is = TestDataBuilder.class
                .getClassLoader().getResourceAsStream(path)) {
            return MAPPER.readValue(is, type);
        } catch (Exception e) {
            throw new RuntimeException("JSON load error: " + path, e);
        }
    }

    public static <T> List<T> fromJsonArray(String path, Class<T> type) {
        try (InputStream is = TestDataBuilder.class
                .getClassLoader().getResourceAsStream(path)) {
            JavaType listType = MAPPER.getTypeFactory()
                                      .constructCollectionType(List.class, type);
            return MAPPER.readValue(is, listType);
        } catch (Exception e) {
            throw new RuntimeException(e);
        }
    }
}

Strategy Pattern
behavioral · auth strategies · environment switching
Intent: Define a family of algorithms, encapsulate each one, and make them interchangeable. In QA: swap auth mechanisms (Bearer, Basic, OAuth2, API Key) at runtime without changing test code.
AuthStrategy.java + implementations Java · RestAssured 
// Strategy interface
public interface AuthStrategy {
    RequestSpecification apply(RequestSpecification spec);
}

// Concrete Strategy 1 — Bearer Token
public class BearerTokenAuth implements AuthStrategy {
    private final String token;

    public BearerTokenAuth(String token) { this.token = token; }

    @Override
    public RequestSpecification apply(RequestSpecification spec) {
        return spec.header("Authorization", "Bearer " + token);
    }
}

// Concrete Strategy 2 — Basic Auth
public class BasicAuth implements AuthStrategy {
    private final String user, pass;

    public BasicAuth(String user, String pass) {
        this.user = user; this.pass = pass;
    }

    @Override
    public RequestSpecification apply(RequestSpecification spec) {
        return spec.auth().basic(user, pass);
    }
}

// Concrete Strategy 3 — API Key
public class ApiKeyAuth implements AuthStrategy {
    private final String key;
    public ApiKeyAuth(String k) { this.key = k; }

    @Override
    public RequestSpecification apply(RequestSpecification spec) {
        return spec.header("x-api-key", key);
    }
}
ApiClient.java — Context that uses Auth Strategy Java · RestAssured 
public class ApiClient {

    private AuthStrategy authStrategy;
    private final String   baseUrl;

    public ApiClient(String baseUrl, AuthStrategy auth) {
        this.baseUrl      = baseUrl;
        this.authStrategy = auth;
    }

    // Switch auth at runtime — no test change needed
    public void setAuth(AuthStrategy auth) { this.authStrategy = auth; }

    public Response get(String path) {
        return authStrategy.apply(RestAssured.given()
            .baseUri(baseUrl).log().ifValidationFails())
            .get(path);
    }

    public Response post(String path, Object body) {
        return authStrategy.apply(RestAssured.given()
            .baseUri(baseUrl).contentType("application/json"))
            .body(body).post(path);
    }
}

Facade Pattern
structural · api helper · db + rest combined operations
Intent: Provide a simplified interface to a complex subsystem. In tests: wrap RestAssured + DB validation + JSON parsing behind a clean PricingApiHelper class — tests read like business requirements.
PricingApiHelper.java — Facade over RestAssured + DB Java · RestAssured · PostgreSQL 
public class PricingApiHelper {

    private final ApiClient  apiClient;
    private final DBHelper   db;
    private final ObjectMapper mapper = new ObjectMapper();

    public PricingApiHelper() {
        this.apiClient = ApiClientFactory.getClient("pricing");
        this.db        = DBHelper.getInstance();
    }

    /** Creates a pricing rule via API and verifies DB persistence */
    public PricingRuleResponse createRuleAndVerify(PricingRuleRequest req) {
        // 1. POST via RestAssured
        Response resp = apiClient.post("/v1/pricing/rules", req);
        resp.then().statusCode(201);

        PricingRuleResponse created = resp.as(PricingRuleResponse.class);

        // 2. Verify persisted in DB (PostgreSQL)
        Map<String, Object> dbRow = db.queryRow(
            "SELECT * FROM pricing_rules WHERE rule_code = ?",
            req.getRuleCode());

        Assert.assertEquals(dbRow.get("power_rank"),
            req.getPowerRank(), "DB power_rank mismatch");

        return created;
    }

    /** Calculates price — wraps complex GET with query params */
    public double calculatePrice(String ruleCode, String vehicleId) {
        return apiClient.get("/v1/pricing/calculate?rule="
                + ruleCode + "&vehicle=" + vehicleId)
            .then().statusCode(200)
            .extract().path("calculatedPrice");
    }

    public void deleteRuleAndVerify(String ruleId) {
        apiClient.delete("/v1/pricing/rules/" + ruleId)
            .then().statusCode(204);
        boolean inDb = db.exists(
            "SELECT 1 FROM pricing_rules WHERE id = ?", ruleId);
        Assert.assertFalse(inDb, "Rule still exists in DB after delete");
    }
}

Decorator Pattern
structural · webdriver wrapping · logging · screenshots · retry
Intent: Attach additional responsibilities to an object dynamically. In Selenium: wrap WebDriver with logging, screenshot-on-failure, and retry behavior without modifying driver code or using inheritance.
LoggingWebDriver.java — Decorator over WebDriver Java · Selenium 
// Decorator — wraps real driver, adds logging to every action
public class LoggingWebDriver implements WebDriver {

    private final WebDriver delegate;
    private static final Logger log =
        LogManager.getLogger(LoggingWebDriver.class);

    public LoggingWebDriver(WebDriver driver) { this.delegate = driver; }

    @Override
    public void get(String url) {
        log.info("→ Navigating to: {}", url);
        delegate.get(url);
        log.info("✓ Page loaded: {}", delegate.getTitle());
    }

    @Override
    public WebElement findElement(By by) {
        log.debug("→ findElement: {}", by);
        try {
            WebElement el = delegate.findElement(by);
            log.debug("✓ Found: {}", by);
            return el;
        } catch (NoSuchElementException e) {
            log.error("✗ Not found: {}", by);
            throw e;
        }
    }

    @Override
    public void quit() {
        log.info("→ Quitting browser");
        delegate.quit();
    }

    // Delegate all other WebDriver methods...
    @Override public String getCurrentUrl() { return delegate.getCurrentUrl(); }
    @Override public String getTitle()       { return delegate.getTitle(); }
    // ... etc
}
RetryFilter.java — RestAssured Filter Decorator (retry on 503) Java · RestAssured 
public class RetryFilter implements Filter {

    private static final int MAX_RETRIES = 3;
    private static final long WAIT_MS    = 1000L;

    @Override
    public Response filter(FilterableRequestSpecification req,
                           FilterableResponseSpecification respSpec,
                           FilterContext ctx) {
        Response resp = null;
        for (int i = 0; i < MAX_RETRIES; i++) {
            resp = ctx.next(req, respSpec);
            if (resp.statusCode() != 503) break;
            System.out.printf("Retry %d/%d after 503%n", i+1, MAX_RETRIES);
            try { Thread.sleep(WAIT_MS * (i + 1)); }
            catch (InterruptedException ie) {
                Thread.currentThread().interrupt();
            }
        }
        return resp;
    }
}

Observer / Listener Pattern
behavioral · testng listeners · screenshot on failure · extent reports
Intent: Define a one-to-many dependency. When test lifecycle events fire (onTestFailure, onStart, onFinish), all registered observers react. In TestNG this is the ITestListener and ISuiteListener system.
TestListener.java — Screenshot + ExtentReport on failure Java · TestNG · Selenium 
public class TestListener implements ITestListener {

    @Override
    public void onTestStart(ITestResult result) {
        ExtentManager.createTest(result.getName(),
            result.getMethod().getDescription());
        System.out.printf("▶ START: %s%n", result.getName());
    }

    @Override
    public void onTestSuccess(ITestResult result) {
        ExtentManager.getTest().pass("✓ PASSED");
    }

    @Override
    public void onTestFailure(ITestResult result) {
        // 1. Capture screenshot
        String path = captureScreenshot(result.getName());

        // 2. Log to Extent Report with screenshot attachment
        ExtentManager.getTest()
            .fail(result.getThrowable())
            .addScreenCaptureFromPath(path, "Failure Screenshot");
    }

    @Override
    public void onTestSkipped(ITestResult result) {
        ExtentManager.getTest().skip(result.getThrowable());
    }

    private String captureScreenshot(String testName) {
        String path = "screenshots/" + testName + "_"
            + System.currentTimeMillis() + ".png";
        TakesScreenshot ts = (TakesScreenshot) DriverManager.getDriver();
        File src = ts.getScreenshotAs(OutputType.FILE);
        try { FileUtils.copyFile(src, new File(path)); }
        catch (IOException e) { e.printStackTrace(); }
        return path;
    }
}

public class RetryAnalyzer implements IRetryAnalyzer {

    private int retryCount = 0;
    private static final int MAX_RETRY = 2;

    @Override
    public boolean retry(ITestResult result) {
        if (retryCount < MAX_RETRY) {
            retryCount++;
            System.out.printf("⟳ Retry %d/%d: %s%n",
                retryCount, MAX_RETRY, result.getName());
            return true;
        }
        return false;
    }
}

Data-Driven Pattern (Template Method)
behavioral · testng @dataprovider · excel (apache poi) · json · jackson
Intent: Separate test logic from test data. The test algorithm stays fixed; only the data varies. Drive tests from Excel sheets (via Apache POI) or JSON files (via Jackson) — same test method, many data rows.
ExcelReader.java — Apache POI Excel → Object[][] Java · Apache POI · TestNG 
public class ExcelReader {

    public static Object[][] readSheet(String filePath, String sheetName) {
        try (FileInputStream fis = new FileInputStream(filePath);
             Workbook          wb  = new XSSFWorkbook(fis)) {

            Sheet sheet = wb.getSheet(sheetName);
            int rows = sheet.getPhysicalNumberOfRows();
            int cols = sheet.getRow(0).getPhysicalNumberOfCells();

            // Row 0 = headers, Row 1+ = data
            Object[][] data = new Object[rows - 1][cols];
            for (int r = 1; r < rows; r++) {
                Row row = sheet.getRow(r);
                for (int c = 0; c < cols; c++) {
                    data[r-1][c] = getCellValue(row.getCell(c));
                }
            }
            return data;

        } catch (Exception e) {
            throw new RuntimeException("Excel read error: " + filePath, e);
        }
    }

    private static Object getCellValue(Cell cell) {
        if (cell == null) return "";
        return switch (cell.getCellType()) {
            case STRING  -> cell.getStringCellValue();
            case NUMERIC -> (long) cell.getNumericCellValue(); // avoid 1.0 → 1
            case BOOLEAN -> cell.getBooleanCellValue();
            case FORMULA -> cell.getStringCellValue();
            default      -> "";
        };
    }
}

Quick Reference Cheat Sheet
all patterns · when to use · which stack
Pattern Category Problem it Solves Where in Your Framework Stack
Singleton Creational Shared resource that must exist only once per thread/run DriverManager, ConfigManager, TokenCache, DBHelper Selenium REST
Factory Method Creational Create objects without specifying exact class; switch impl via config DriverFactory (chrome/firefox/edge), ApiClientFactory (service URL switching) Selenium REST
Builder Creational Construct complex objects step by step with optional fields PricingRuleRequest.Builder, TestDataBuilder from JSON/Excel REST JSON Excel
Page Object Model Structural Decouple UI locators from test logic; single change point LoginPage, DashboardPage, RulesConfigPage extend BasePage Selenium TestNG
Facade Structural Simplify complex subsystem — hide RestAssured + DB behind clean API PricingApiHelper, AuthHelper, TestSetupHelper REST TestNG
Decorator Structural Add behavior (logging, retry, screenshot) without changing original class LoggingWebDriver, RetryFilter, TimingFilter for RestAssured Selenium REST
Strategy Behavioral Swap algorithms (auth schemes) at runtime without changing callers BearerTokenAuth, BasicAuth, ApiKeyAuth — swapped per env in config REST Java
Observer / Listener Behavioral React to lifecycle events (test start/fail/finish) without polling TestListener, RetryAnalyzer, SuiteListener for ExtentReports TestNG Selenium
Data-Driven (Template) Behavioral Same test logic, many data sets — decouple data from code @DataProvider reading Excel via POI or JSON via Jackson TestNG Excel JSON
Recommended Pattern Combinations

🔒 Parallel Selenium Suite

  • ThreadLocal Singleton → DriverManager
  • Factory → DriverFactory (browser from -Dbrowser param)
  • POM → Page classes extend BasePage
  • Observer → TestListener (screenshot + Extent)
  • Data-Driven → Excel @DataProvider for UI test data

🌐 RestAssured API Suite

  • Singleton → ConfigManager + TokenCache
  • Factory → ApiClientFactory per microservice
  • Builder → Request payload objects
  • Strategy → Auth switching per env
  • Facade → PricingApiHelper (REST + DB)
  • Decorator → RetryFilter + TimingFilter
  • Data-Driven → JSON @DataProvider
