üí° Interview Tips & Best Practices

  • Encapsulation: Use private for fields, provide public getters/setters
  • Class-level: Top-level classes can only be public or default (no protected/private)
  • Constructors: Private constructors prevent instantiation (Singleton pattern)
  • Interface members: Always public by default (methods & constants)
  • Local variables: Cannot have access modifiers (only method parameters and class members)
  • Protected caveat: Subclass in different package accesses protected via inheritance, not instance reference
  • Default choice: Start with most restrictive, increase visibility only as needed
  • Common mistake: Forgetting that protected allows same package access (not just subclasses)
