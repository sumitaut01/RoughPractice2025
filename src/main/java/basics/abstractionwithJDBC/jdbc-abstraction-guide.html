What is Abstraction in JDBC?

Abstraction in JDBC means hiding database-specific implementation details behind a unified interface.

You write the same Java code whether connecting to MySQL, PostgreSQL, Oracle, or any other database.

When asked about JDBC abstraction, emphasize "Program to an interface, not an implementation" - this allows vendors to provide their own implementations while developers use a standard API.

1. DriverManager

What is DriverManager?

DriverManager is a class that manages JDBC drivers and acts as a factory for creating database connections.

Key Responsibilities

  • Driver Registration: Maintains list of available JDBC drivers
  • Connection Creation: Selects appropriate driver and establishes connections
  • Driver Selection: Automatically picks right driver based on URL

Abstraction in Action

// Step 1: Register driver (automatic in JDBC 4.0+)
Class.forName("com.mysql.cj.jdbc.Driver");

// Step 2: DriverManager abstracts driver selection
Connection conn = DriverManager.getConnection(
    "jdbc:mysql://localhost:3306/mydb",
    "username",
    "password"
);

// You work with Connection interface, not vendor classes!
Key Point: You never instantiate driver-specific classes like com.mysql.cj.jdbc.ConnectionImpl. DriverManager returns the standard Connection interface.
DriverManager uses the connection URL to determine which driver to use. The URL format (jdbc:mysql, jdbc:postgresql) acts as a selection mechanism.

2. Connection Interface

What is Connection?

Connection is an interface representing a database session. It provides methods for statements, transactions, and metadata.

Why an Interface?

üîå Pluggability

Each vendor provides their implementation

üîÑ Flexibility

Switch databases without code changes

üìù Standard API

Consistent methods across databases

üéØ Decoupling

No dependency on vendor classes

Behind the Scenes

// You program to the Connection interface
Connection conn = DriverManager.getConnection(url, user, pass);

// Behind the scenes:
// MySQL: returns com.mysql.cj.jdbc.ConnectionImpl
// PostgreSQL: returns org.postgresql.jdbc.PgConnection
// Oracle: returns oracle.jdbc.driver.OracleConnection

// But you only use the Connection interface!
Statement stmt = conn.createStatement();
PreparedStatement pstmt = conn.prepareStatement("SELECT * FROM users");
conn.commit();
conn.close();

3. getConnection() Method

getConnection() is a static DriverManager method that establishes database connections.

Method Signatures

// URL only
Connection getConnection(String url)

// URL with credentials
Connection getConnection(String url, String user, String password)

// URL with properties
Connection getConnection(String url, Properties info)

How It Provides Abstraction

Process:
  1. Call getConnection() with database URL
  2. DriverManager iterates registered drivers
  3. Each driver checks if it handles the URL
  4. First compatible driver creates connection
  5. Returns Connection interface (hiding implementation)

Database-Independent Code

public class DatabaseManager {
    public Connection getConnection(String dbType) throws SQLException {
        String url, user, password;
        
        switch(dbType) {
            case "mysql":
                url = "jdbc:mysql://localhost:3306/mydb";
                user = "root";
                password = "password";
                break;
            case "postgresql":
                url = "jdbc:postgresql://localhost:5432/mydb";
                user = "postgres";
                password = "password";
                break;
            default:
                throw new IllegalArgumentException("Unknown database");
        }
        
        // Same call works for any database!
        return DriverManager.getConnection(url, user, password);
    }
}
The URL format acts as a service locator pattern. The prefix tells DriverManager which driver to use.

Interview Questions

Q1: Why is Connection an interface?

Answer: To provide abstraction and allow vendors to implement their own versions while developers use a standard API. This enables database independence.

Q2: How does DriverManager select the right driver?

Answer: It iterates through registered drivers and calls their acceptsURL() method. The first driver returning true is used. The URL format determines the match.

Q3: DriverManager vs DataSource?

Answer: DriverManager is basic, suitable for simple apps. DataSource supports connection pooling and distributed transactions - preferred for enterprise apps.

Q4: Complete abstraction flow?

  1. App calls DriverManager.getConnection(url)
  2. DriverManager selects driver based on URL
  3. Driver returns vendor-specific Connection
  4. App receives Connection interface reference
  5. App uses standard Connection methods
  6. Vendor implementation executes behind interface

Q5: What if no driver found?

Answer: DriverManager throws SQLException: "No suitable driver found for [url]". Either driver JAR missing from classpath or URL format incorrect.

Complete Example

import java.sql.*;

public class JDBCDemo {
    private static final String URL = "jdbc:mysql://localhost:3306/school";
    private static final String USER = "root";
    private static final String PASS = "password";
    
    public static void main(String[] args) {
        // Load driver (automatic in JDBC 4.0+)
        try {
            Class.forName("com.mysql.cj.jdbc.Driver");
        } catch (ClassNotFoundException e) {
            System.out.println("Driver not found!");
            return;
        }
        
        // Get connection - abstraction in action
        try (Connection conn = DriverManager.getConnection(URL, USER, PASS)) {
            
            // Create statement
            Statement stmt = conn.createStatement();
            ResultSet rs = stmt.executeQuery("SELECT * FROM students");
            
            while (rs.next()) {
                System.out.printf("ID: %d, Name: %s%n", 
                    rs.getInt("id"), rs.getString("name"));
            }
            
            // Prepared statement
            PreparedStatement pstmt = conn.prepareStatement(
                "INSERT INTO students (name, age) VALUES (?, ?)");
            pstmt.setString(1, "John");
            pstmt.setInt(2, 20);
            pstmt.executeUpdate();
            
            // Transaction
            conn.setAutoCommit(false);
            conn.commit();
            
            rs.close();
            stmt.close();
            pstmt.close();
            
        } catch (SQLException e) {
            e.printStackTrace();
        }
    }
}
Notice: Switch to PostgreSQL/Oracle by changing:
  • Driver class in Class.forName()
  • URL string
  • Credentials
Everything else stays the same!
