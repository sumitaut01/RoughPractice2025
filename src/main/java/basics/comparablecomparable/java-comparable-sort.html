01

What is Natural Order?

In Java, Natural Order means the default way objects of a class sort themselves. Built-in types already have it baked in:

Integer → ascending (1, 2, 3) String → lexicographic (a, b, c) Double → ascending (1.1, 2.2) LocalDate → chronological char → Unicode order

For your custom classes (like Employee, Student), Java doesn't know the order unless YOU define it via Comparable.

02

The Comparable Interface

INTERFACE java.lang.Comparable<T>

It lives in java.lang — no import needed. It has exactly ONE method to implement:

public interface Comparable<T> {
    int compareTo(T other);
}
💡

Mental Model: Think of compareTo() as asking:
"Am I (this) less than, equal to, or greater than other?"

HOW TO USE

Your class implements Comparable:

class Student implements Comparable<Student> {

    int marks;

    @Override
    public int compareTo(Student other) {
        return this.marks - other.marks;
        // ascending by marks
    }
}

WHAT UNLOCKS

Once you implement it, ALL of these work:

Collections.sort(studentList);

Arrays.sort(studentArray);

new TreeSet<>(studentList);

new PriorityQueue<>(studentList);

student1.compareTo(student2);
// all use your compareTo!
03

The compareTo() Contract

The return value of compareTo() tells Java where to place THIS object relative to OTHER:

< 0
THIS comes
BEFORE other
= 0
EQUAL
(same position)
> 0
THIS comes
AFTER other
⚡

Memory Trick: Think of it as subtraction.
this.age - other.age → ascending (small → big)
other.age - this.age → descending (big → small)

CONCRETE EXAMPLE Integer comparison flow

a = 5
→
compareTo(b=9)
→
5 - 9 = -4
→
5 comes BEFORE 9
Integer a = 5, b = 9;
System.out.println(a.compareTo(b));  // -1 (negative → a before b)
System.out.println(b.compareTo(a));  // 1  (positive → b after a)
System.out.println(a.compareTo(5)); // 0  (equal)
04

Real-World Examples

EXAMPLE 1 Student — sort by marks

import java.util.*;

class Student implements Comparable<Student> {

    String name;
    int    marks;

    Student(String name, int marks) {
        this.name  = name;
        this.marks = marks;
    }

    @Override
    public int compareTo(Student other) {
        return this.marks - other.marks; // ascending
    }

    @Override
    public String toString() {
        return name + "(" + marks + ")";
    }
}

public class Main {
    public static void main(String[] args) {

        List<Student> list = new ArrayList<>();
        list.add(new Student("Riya",  85));
        list.add(new Student("Sumit", 92));
        list.add(new Student("Arjun", 78));

        Collections.sort(list); // uses compareTo()

        System.out.println(list);
        // Output: [Arjun(78), Riya(85), Sumit(92)]
    }
}
✅

Output: [Arjun(78), Riya(85), Sumit(92)] — sorted ascending by marks.

EXAMPLE 2 Employee — sort by salary, then name

class Employee implements Comparable<Employee> {

    String name;
    double salary;

    Employee(String name, double salary) {
        this.name   = name;
        this.salary = salary;
    }

    @Override
    public int compareTo(Employee other) {
        // Primary: salary ascending
        int result = Double.compare(this.salary, other.salary);

        // Secondary: if salary equal → sort by name
        if (result == 0) {
            result = this.name.compareTo(other.name);
        }
        return result;
    }
}
💡

Tip: Use Double.compare(a, b) instead of a - b for doubles/floats to avoid floating-point overflow bugs.

EXAMPLE 3 String already implements Comparable

List<String> names = Arrays.asList("Zebra", "Apple", "Mango");
Collections.sort(names);         // lexicographic (dictionary) order
System.out.println(names);       // [Apple, Mango, Zebra]

// Behind the scenes, String.compareTo() compares char by char:
"Apple".compareTo("Mango");    // negative → Apple before Mango
"Zebra".compareTo("Apple");    // positive → Zebra after Apple

EXAMPLE 4 TreeSet uses compareTo() automatically

TreeSet<Student> set = new TreeSet<>();
set.add(new Student("Riya",  85));
set.add(new Student("Sumit", 92));
set.add(new Student("Arjun", 78));

// TreeSet maintains sorted order using compareTo()
System.out.println(set);
// Output: [Arjun(78), Riya(85), Sumit(92)]

// PriorityQueue also uses compareTo() for heap ordering
PriorityQueue<Student> pq = new PriorityQueue<>(set);
System.out.println(pq.poll()); // Arjun(78) ← lowest first
⚠️

TreeSet without Comparable = Runtime crash! If your class doesn't implement Comparable and you haven't provided a Comparator, TreeSet throws ClassCastException at runtime.

05

Comparable vs Comparator

Aspect Comparable Comparator
Package java.lang java.util
Method compareTo(T o) compare(T o1, T o2)
Who defines it? The class itself External / separate class
Order types Only ONE natural order Multiple orders possible
Modifies class? Yes (implements interface) No (external)
Used by Collections.sort, TreeSet, PriorityQueue Collections.sort(list, cmp), TreeSet(cmp)
Best for Default/natural sort Multiple/custom/ad-hoc sorts

COMPARABLE Single natural order

class Student
    implements Comparable<Student> {

    @Override
    public int compareTo(Student o) {
        return this.marks - o.marks;
    }
}
// Always sorts by marks
Collections.sort(list);

COMPARATOR Flexible, multiple orders

// Sort by name
Comparator<Student> byName =
    (a, b) -> a.name.compareTo(b.name);

// Sort by marks descending
Comparator<Student> byDesc =
    (a, b) -> b.marks - a.marks;

Collections.sort(list, byName);
Collections.sort(list, byDesc);
🧠

Rule of thumb: Use Comparable when there's one obvious natural order ("students always sort by marks"). Use Comparator when you need multiple sort strategies or can't modify the class.

06

sort() — Everywhere in Java

Collections.sort() — for Lists

List<Integer> nums = Arrays.asList(3, 1, 4, 1, 5, 9);

// Natural order (uses compareTo)
Collections.sort(nums);
System.out.println(nums); // [1, 1, 3, 4, 5, 9]

// Reverse order
Collections.sort(nums, Collections.reverseOrder());
System.out.println(nums); // [9, 5, 4, 3, 1, 1]

// Same as list.sort() in Java 8+
nums.sort(Comparator.naturalOrder());

Arrays.sort() — for arrays

int[] arr = {5, 2, 8, 1, 9};
Arrays.sort(arr);
// Output: [1, 2, 5, 8, 9]

String[] words = {"Banana", "Apple", "Cherry"};
Arrays.sort(words);
// Output: [Apple, Banana, Cherry]

// Partial array sort (index 1 to 3)
Arrays.sort(arr, 1, 4);

// Object array with Comparator
Arrays.sort(words, Comparator.reverseOrder());
// Output: [Cherry, Banana, Apple]

Stream.sorted() — Java 8+ streams

List<Student> sorted = studentList.stream()
    .sorted()                         // uses compareTo()
    .collect(Collectors.toList());

// Or with custom comparator
List<Student> byName = studentList.stream()
    .sorted(Comparator.comparing(s -> s.name))
    .collect(Collectors.toList());

// Chained: by marks desc, then name asc
List<Student> complex = studentList.stream()
    .sorted(Comparator.comparingInt(Student::marks)
                      .reversed()
                      .thenComparing(s -> s.name))
    .collect(Collectors.toList());

Comparator.comparing() — Modern Java 8 way

// Instead of writing lambdas, use factory methods:

list.sort(Comparator.comparing(Employee::getName));

list.sort(Comparator.comparingDouble(Employee::getSalary));

// Chain multiple: salary asc, then name asc
list.sort(Comparator.comparingDouble(Employee::getSalary)
                   .thenComparing(Employee::getName));

// Reverse a Comparable's natural order
list.sort(Comparator.reverseOrder());
list.sort(Comparator.naturalOrder());

Gotchas & Tips

GOTCHA 1 Integer overflow in compareTo()

// ❌ WRONG — can overflow for large negatives
return this.value - other.value;

// ✅ SAFE — no overflow risk
return Integer.compare(this.value, other.value);

// Why? Integer.MIN_VALUE - 1 wraps to MAX_VALUE → wrong sort!
🚨

Always use Integer.compare(a, b), Double.compare(a, b) for numeric comparisons — NOT subtraction.

GOTCHA 2 Consistency with equals()

// If compareTo returns 0, equals() should return true
// TreeSet uses compareTo, NOT equals — inconsistency = bugs!

class Product implements Comparable<Product> {
    String name; int id;

    public int compareTo(Product o) {
        return this.name.compareTo(o.name); // by name
    }

    public boolean equals(Object o) {
        return this.id == ((Product)o).id; // by id ← MISMATCH!
    }
}
// TreeSet: two Products with same name but different ids
// will be treated as DUPLICATES (compareTo=0) → data loss!

TIP Null-safe comparisons

// Handle nulls in Comparator
list.sort(Comparator.nullsFirst(
    Comparator.comparing(Employee::getName)
));

list.sort(Comparator.nullsLast(
    Comparator.comparing(Employee::getSalary)
));
