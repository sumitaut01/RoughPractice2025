TestNG Advanced Study Guide

This guide covers dynamic test suite generation, the @Factory annotation, and data-driven execution.

1. Generating testng.xml at Runtime

Useful when you don't know which tests to run until the application starts (e.g., based on a database flag).

Programmatic Suite Creation

XmlSuite suite = new XmlSuite();
suite.setName("DynamicSuite");

XmlTest test = new XmlTest(suite);
test.setName("ParameterTest");

List<XmlClass> classes = new ArrayList<>();
classes.add(new XmlClass("com.tests.LoginTest"));
test.setXmlClasses(classes);

TestNG tng = new TestNG();
tng.setXmlSuites(Arrays.asList(suite));
tng.run();

2. The @Factory Annotation

A Factory produces multiple instances of a test class. Each instance acts as its own separate "world" with its own state.

Key Differences

Feature @DataProvider @Factory
Target Method-level Class-level
Behavior Runs one method multiple times Creates multiple class objects
State Shared variables can cause conflicts Each instance has private variables

3. Data-Driven Factory (The Hybrid Approach)

Combining @DataProvider with @Factory allows you to read from external sources (Excel/JSON) to spawn test instances.

The Template Class

public class BrowserTest {
    private String browser;
    
    public BrowserTest(String browser) { this.browser = browser; }

    @Test
    public void testLogo() {
        System.out.println("Testing logo on: " + browser);
    }
}

The Factory Logic

public class MyFactory {
    @Factory(dataProvider = "source")
    public Object[] create(String b) {
        return new Object[] { new BrowserTest(b) };
    }

    @DataProvider
    public Object[][] source() {
        // Replace with Excel/JSON reading logic
        return new Object[][] { {"Chrome"}, {"Firefox"}, {"Edge"} };
    }
}
