01 — The Problem

Why SoftAssert Can Betray You

SoftAssert is designed to collect multiple failures and report them all at once — great for validating multiple fields in an API response. But there's a critical gotcha that silently corrupts your test results.

⚠ The Silent Failure

If you call soft.assertEquals() and forget to call soft.assertAll() at the end of the test — your test PASSES, even if assertions failed. TestNG has no idea. The failures are swallowed.

🔇

Silent Failures

Assertion failures are collected inside the SoftAssert object but never thrown. The test method ends normally. Green tick. Everything looks fine.

🎭

False Confidence

Your pipeline is green. The team ships. But bugs that were caught by assertions are silently ignored because assertAll() was missing.

🔀

Hard to Track

In parallel test runs with 30+ tests, figuring out which tests forgot assertAll() is a manual nightmare. A review comment doesn't scale.

⚡

Human Error

Under deadline pressure, test methods get written fast. A missing assertAll() is not caught by any compiler. It's invisible to code review tools.

BuggyTest.java
❌ Silent Bug
@Test
public void validatePricingRule() {
    SoftAssert soft = new SoftAssert();

    soft.assertEquals(response.getStatusCode(), 200, "Status mismatch");
    soft.assertNotNull(response.jsonPath().get("ruleId"), "ruleId is null");
    soft.assertEquals(response.jsonPath().get("powerRank"), 5, "Power rank wrong");

    // 😱 forgot assertAll() — test PASSES even if all 3 assertions above FAILED
}
02 — The Solution

A Listener That Never Forgets

We implement IInvokedMethodListener — a TestNG hook that fires after every @Test method. The listener holds a ThreadLocal<SoftAssert> and calls assertAll() automatically, no matter what.

✅ The Guarantee

Once this pattern is in place, it is impossible for a soft assertion failure to go unreported — even if every single developer on the team forgets to write assertAll().

🧪

Test starts

Listener's beforeInvocation fires. It clears any stale SoftAssert from the ThreadLocal — fresh slate for this test.

📝

Test runs assertions

Test calls SoftAssertContext.create() which creates a new SoftAssert and registers it in the ThreadLocal. All soft.assertXxx() calls are collected internally.

⚙️

Test method returns

Doesn't matter if assertAll() was called or not. The test method exits.

🛡️

Listener's afterInvocation fires

Fetches the SoftAssert from ThreadLocal and calls assertAll(). If any assertions failed, it catches the AssertionError, marks the test as FAILURE, and attaches the error message — exactly as if you had called it yourself.

03 — The Code

Full Implementation

Three files. Drop them into your framework and you're done.

1

ThreadLocal SoftAssert Context

This class is the glue. It stores each thread's SoftAssert instance, making the pattern safe for parallel TestNG execution.

SoftAssertContext.java
Core Utility
package com.yourcompany.framework.context;

import org.testng.asserts.SoftAssert;

/**
 * ThreadLocal holder for SoftAssert instances.
 * Ensures parallel test threads never share assertion state.
 */
public class SoftAssertContext {

    private static final ThreadLocal<SoftAssert> HOLDER = new ThreadLocal<>();

    /** Create a new SoftAssert and register it for this thread. */
    public static SoftAssert create() {
        SoftAssert sa = new SoftAssert();
        HOLDER.set(sa);
        return sa;
    }

    /** Retrieve the current thread's SoftAssert (may be null). */
    public static SoftAssert get() {
        return HOLDER.get();
    }

    /** Always call this after use to prevent memory leaks. */
    public static void clear() {
        HOLDER.remove();
    }
}
2

The Listener — The Safety Net

Implements IInvokedMethodListener to intercept every test method. Calls assertAll() in afterInvocation — guaranteed execution.

SoftAssertListener.java
✅ The Safety Net
package com.yourcompany.framework.listeners;

import com.yourcompany.framework.context.SoftAssertContext;
import org.testng.*;
import org.testng.asserts.SoftAssert;

public class SoftAssertListener implements IInvokedMethodListener {

    @Override
    public void beforeInvocation(IInvokedMethod method, ITestResult testResult) {
        if (method.isTestMethod()) {
            // Wipe any leftover state from a previous test on this thread
            SoftAssertContext.clear();
        }
    }

    @Override
    public void afterInvocation(IInvokedMethod method, ITestResult testResult) {
        if (!method.isTestMethod()) return;

        SoftAssert softAssert = SoftAssertContext.get();
        if (softAssert == null) return; // test didn't use SoftAssert — skip

        try {
            softAssert.assertAll(); // 🛡️ the safety net
        } catch (AssertionError e) {
            // Mark test as FAILED and attach the full failure message
            testResult.setStatus(ITestResult.FAILURE);
            testResult.setThrowable(e);
        } finally {
            // Always clean up — prevent memory leaks in long parallel runs
            SoftAssertContext.clear();
        }
    }
}
3

Register the Listener

One entry in your testng.xml activates protection for the entire suite. Or use @Listeners on your BaseTest.

testng.xml
Config
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE suite SYSTEM "http://testng.org/testng-1.0.dtd">

<suite name="X4V-Suite" parallel="methods" thread-count="5">

    <listeners>
        <listener class-name="com.yourcompany.framework.listeners.SoftAssertListener"/>
    </listeners>

    <test name="Pricing Rules">
        <classes>
            <class name="com.yourcompany.tests.PricingRuleTests"/>
        </classes>
    </test>
</suite>

Same test scenario. Same assertions. Drastically different outcomes without the safety net.

❌ Without Listener

public void validateRule() {

✅ With Listener

public void validateRule() {
  SoftAssert soft =
      SoftAssertContext.create();

  // These 2 assertions FAIL:
  soft.assertEquals(
    status, 200, "Status wrong");
  soft.assertNotNull(
    ruleId, "ruleId null");

  // forgot assertAll() — doesn't matter
  // Listener calls it automatically

  // Result: ❌ TEST FAILS (correctly)
}

05 — Migration Checklist

Adopting This in Your Project

1

Add SoftAssertContext.java

Create the SoftAssertContext class in your framework/context package.

2

Add SoftAssertListener.java

Create the listener class in your framework/listeners package.

3

Register in testng.xml

Add <listener class-name="...SoftAssertListener"/> once — protects all tests in the suite.

4

Replace new SoftAssert() with SoftAssertContext.create()

Do a project-wide find-and-replace. Any test that still uses new SoftAssert() will not be tracked by the listener.

5

(Optional) Remove redundant assertAll() calls

They're harmless to keep — calling assertAll() twice is safe and idempotent if no new assertions were added. Keeping them improves readability.

One Listener. Total Coverage.

This pattern is the difference between a framework that relies on developer discipline and one that enforces correctness by design. Add it once, and your entire test suite is protected — now and for every test written in the future.
